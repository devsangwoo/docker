- 하나 하나가 서버/프론트/백앤드-> 이걸 트리로 묶은 노드
  --도커이미지화.
  --이미지를 보고 컨테이너 생성.

- 컨테이너를 만든 순간 그 자체로 존재함, 도커를 내리면 컨테이너는 없어짐. 휘발성.

- 이미지의 최신화가 가장 중요함
- 파일질라, FTP로 STFP 파일전송을 서버와 연동
- 서버 자체를 날리고, 이미지를 보고 다시 생성함.
- 개발쪽 <-> 서버쪽 환경이 다름
- 한쪽이 버전업이 되면, 로컬에도 1:1적으로 맞춰줘야함
- 도커위에 환경을 만듬.
- OS -> Docker -> 환경
- 이체부분 컨테이너, 뱅킹 부분 컨테이너, 조회 부분 컨테이너 나눠서 쿠버네티스(스완)
